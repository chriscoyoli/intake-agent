Universal Intake Agent

One front door in Slack for every employee request

PROOF OF CONCEPT · RUNNING LIVE

The problem

Every team has a different intake path: Legal by email, IT by ticket, Marketing by form, Product by spreadsheet. Employees must know which door to use and fill out a different thin form each time. Those forms arrive incomplete, triggering rounds of clarifying questions, and once submitted, the requester has no visibility into what happens next.

The approach

An employee types /problem in Slack and describes the need in plain language. An AI agent interprets it, asks only the few qualifying questions still missing, and assembles one complete, well-formed request. It routes that request into the tool the destination team already uses, so the employee never learns a new system, and sends them plain-language status updates as the work moves.

Running today

  • Live Slack bot, powered by Claude, hosted always-on.
  • Real routing into Airtable, Jira, Asana, plus a Google Sheet as the unified record.
  • Automatic status breadcrumbs back to the requester.

Why it generalizes

  • Every tool sits behind one small interface.
  • One structured request, rendered into each team's format.
  • Any team with an inbound queue plugs in the same way.

Path to production

  • This POC uses only synthetic test data.
  • For real data, the same design deploys inside the company's own cloud and compliance boundary.
  • Data flows one direction (tool to record); no new processor required.

Built as a working proof of concept. Slack is the only surface employees see; each team keeps using the tool it already has; one Google Sheet holds every request across every team.
